# ShramSetu – Production Architecture & Tech Stack

## 🧠 Problem Statement

India has over 380 million informal workers (~92% of workforce) who:
- Lack digital identity and visibility
- Are exploited by middlemen and underpaid
- Do not have access to structured pricing
- Are excluded from welfare schemes and financial systems

As seen in personas like Manoj (urban electrician) and Palak (rural tailor), 
workers rely on word-of-mouth, cash payments, and lack stability. :contentReference[oaicite:0]{index=0}

## 🎯 Vision

Build a unified digital ecosystem that:
- Gives workers a verified digital identity
- Enables direct service booking
- Standardizes pricing and trust (ratings, reviews)
- Integrates UPI payments
- Provides access to loans, insurance, and schemes

## 🚀 Core Differentiation

similer webpages-> urban company, shramin

- Focus on **service delivery marketplace**, not just hiring
- Workers act as:
  - Freelancers (gig-based)
  - Interns (learning + earning)
  - Full-time professionals (top-tier workers)
- Financial inclusion layer (loans, benefits)
- Hybrid online + field agent onboarding model

# 🏗️ Tech Stack (Production Level)

## 🖥️ Frontend
- Framework: Next.js (React + SSR)
- Styling: Tailwind CSS + ShadCN UI
- State Management: Zustand / Redux Toolkit
- Forms: React Hook Form + Zod
- Maps: Mapbox / Google Maps API
- Auth UI: Clerk / Firebase Auth

## ⚙️ Backend
- Runtime: Node.js (TypeScript)
- Framework: NestJS (preferred) OR Express (simpler start)
- API: REST (Phase 1) → GraphQL (later scaling)
- Auth:
  - JWT + Refresh Tokens
  - OAuth (Google, Phone OTP)
- File Uploads: Cloudinary / AWS S3
- Payment:
  - Razorpay (UPI, cards)
  - Stripe (future global)

## 🗄️ Database
- Primary DB: PostgreSQL
- ORM: Prisma
- Cache: Redis (sessions, search caching)
- Search Engine: Elasticsearch / Meilisearch (for workers)

## 📱 Mobile (Future Phase)
- React Native / Flutter

## ☁️ DevOps & Infra
- Hosting: AWS / GCP / Vercel (frontend)
- Backend: AWS EC2 / ECS / Railway
- Database: AWS RDS / Supabase
- CDN: Cloudflare
- CI/CD: GitHub Actions
- Monitoring: Prometheus + Grafana / Sentry

# 🔐 Security Considerations

- Aadhaar/KYC verification (later phase)

- Introduce queue system (BullMQ / Kafka)

## 🔄 Core User Flow

Client Flow:
1. Open website
2. Search service (e.g., electrician)
3. View nearby workers
4. Compare profiles (ratings, pricing)
5. Book service
6. Pay via UPI
7. Leave review

Worker Flow:
1. Sign up via phone OTP
2. Create profile (skills, pricing, location)
3. Get verified (manual/agent)
4. Receive booking requests
5. Accept/reject job
6. Complete work
7. Get paid
